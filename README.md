# Online-Food-Delivery-Order-Analysis
This SQL mini project focuses on analyzing online food delivery data similar to platforms like Swiggy or Zomato.
The goal of the project is to extract meaningful business insights from customer orders, restaurant performance, and food item sales using SQL queries.

The project uses multiple relational tables such as customers, restaurants, orders, and order_items to simulate a real-world food delivery database.
By applying SQL joins, aggregate functions, subqueries, and date-based analysis, the project answers key business questions like customer behavior, popular restaurants, peak ordering periods, and revenue trends.

This project helps understand how SQL is used in real companies for data analysis and decision-making.

🔹 Key Objectives

Analyze customer ordering patterns

Identify top-performing restaurants

Find most ordered food items

Calculate total revenue and average order value

Analyze daily and monthly order trends

🔹 Skills Used

SQL Joins (INNER JOIN)

Aggregate Functions (SUM, COUNT, AVG)

GROUP BY & HAVING

Date functions

Subqueries

🔹 Tools

SQL (MySQL)
